**Step 2: Create PhotoUploadService class**
Implement with dependency injection of PDO and ListingPhotoRepository:

**Constructor:**
- __construct(PDO $pdo, ListingPhotoRepository $photoRepo)
- Optionally inject config constants: MAX_UPLOAD_SIZE_MB (default 5), MAX_PHOTOS_PER_LISTING (default 10), IMAGE_MAX_DIMENSION (default 8000), IMAGE_MIN_DIMENSION (default 640), JPEG_QUALITY (default 90)

**Methods:**

- validatePhoto($file): Returns [is_valid, errors]
  - Check MIME type using finfo: image/jpeg, image/png, image/webp only
  - Check file size: 1B to 5MB
  - Check dimensions after loading image: min 640×480, max 8000×8000
  - Check magic bytes (file header matches MIME type)
  - Return validation errors array if any check fails

- reencodeAndStore($file, $userId, $listingId, $displayOrder): Returns photo_url or throws exception
  - Create directories recursively: public/uploads/users/{$userId}/listings/{$listingId}/
  - Generate filename: {timestamp}_{random}.{ext} (e.g., 1726918234_a1b2c3.jpg)
    - timestamp: time() for creation tracking
    - random: substr(md5(uniqid()), 0, 6) for security
    - ext: extract from MIME type or $file['name']
  - Load image via intervention/image: $image = Image::make($file['tmp_name'])
  - Re-encode to strip EXIF:
    - $image->resize(null, 2000, function($constraint) { $constraint->aspectRatio(); })  // Max 2000px height
    - $image->orientate()  // Fix rotation
    - $image->save($destinationPath)  // JPEG save automatically strips metadata
  - Store relative path in database via $photoRepo->create([listing_id, photo_url, display_order])
  - Return photo_url or full URL for response

- deletePhoto($photoId, $userId): Deletes file from filesystem and soft-deletes DB record
  - Verify ownership (check listing.seller_id == $userId)
  - Delete file from public/uploads/
  - Call $photoRepo->delete($photoId) for soft-delete

**Error handling:**
- Throw InvalidArgumentException for validation failures (caught in controller as 422)
- Throw RuntimeException for file system errors (caught as 500)
- Use try-catch around intervention/image calls

**Constants defined via .env** (update .env.example):
- UPLOAD_DIR=public/uploads
- MAX_UPLOAD_SIZE_MB=5
- MAX_PHOTOS_PER_LISTING=10
- IMAGE_MAX_DIMENSION=8000
- IMAGE_MIN_DIMENSION=640
- JPEG_QUALITY=90

**Step 1: Update ListingController with uploadPhotos method**

Add uploadPhotos($get, $post, $files, $id) method (protected endpoint):
- Extract listing_id from $id
- Verify listing exists and user owns it (seller_id == current user)
- Extract files array from $_FILES['photos'] (multipart form with multiple files)
- Check photo count: If count($_FILES['photos']['tmp_name']) > MAX_PHOTOS, return Response::error('Maximum 10 photos per listing', 409)
- Check existing photos: Query current photo count; if count + new_files > 10, return 409
- Iterate each file:
  - Call $photoUploadService->validatePhoto($file) → return 422 if validation fails
  - Call $photoUploadService->reencodeAndStore($file, $userId, $listingId, $displayOrder) where $displayOrder = current_count + index
- Update listing.photo_count via service (for denormalization/performance)
- Return Response::success(['photos_uploaded' => count($files)], 201) with photo URLs

**Step 2: Add uploadAvatar method to handle user profile photos**

Add uploadAvatar($get, $post, $files) method (protected endpoint):
- Extract current user_id from $_SESSION
- Check single file from $_FILES['avatar']
- Call $photoUploadService->validatePhoto($file)
- Call $photoUploadService->reencodeAndStore($file, $userId, null, null) (no listing_id for user avatar)
  - Store to public/uploads/users/{$user_id}/ (not in listings subdirectory)
- Update users.avatar_url via UserRepository
- Return Response::success(['avatar_url' => $url], 201)

**Step 4: Create .htaccess for upload directory security**

Create `public/uploads/.htaccess`:
```apache
<FilesMatch "\.php$">
    Order Deny,Allow
    Deny from all
</FilesMatch>
<FilesMatch "\.php5$">
    Order Deny,Allow
    Deny from all
</FilesMatch>
```

This prevents PHP execution in upload directory (critical security).

**Step 5: Create public/uploads directories**
