<task type="auto">
  <name>Task 2: Create ListingService with validation and business logic</name>
  <files>src/Services/ListingService.php</files>
  <action>
Create ListingService class with dependency injection of PDO, ListingRepository, ListingPhotoRepository, and GeolocationService:

**Core methods:**
- __construct(PDO $pdo, ListingRepository $listingRepo, ListingPhotoRepository $photoRepo, GeolocationService $geolocation)
- createListing($data, $userId): Returns listing_id. Validates required fields (title, category_id, price, address, condition). Geocodes address via GeolocationService. Stores coordinates in latitude/longitude columns. Returns 400 if geocoding fails.
- updateListing($listingId, $data, $userId): Validates ownership (seller_id == $userId). Updates only permitted fields. Returns 403 if not owner.
- deleteListing($listingId, $userId): Soft-deletes listing (sets deleted_at). Returns 403 if not owner.
- getListingById($listingId): Returns full listing with photos (uses LEFT JOIN to avoid N+1). Filters deleted listings.
- listAllListings($filters = [], $limit = 20, $offset = 0): Returns paginated active listings with photos. Accepts category_id, status, price_min, price_max filters.

**Validation rules (use respect/validation):**
- title: Required, string, 10-255 chars
- description: Required, string, 20-5000 chars
- category_id: Required, integer, must exist in categories table
- price: Required, decimal, 0.01-999999.99
- condition: Required, enum ['Excellent', 'Good', 'Fair', 'Poor']
- address: Required, object with street, city, province/state, postal_code, country (each required, 2-100 chars)
- brand, model, year, accessories: Optional (allow NULL)

Return validation errors as Response::validationErrors($errors, 422) on failure.

All database operations use prepared statements. Follow Phase 1-2 service patterns.

<task type="auto">
  <name>Task 3: Create ListingController and wire routes</name>
  <files>src/Controllers/ListingController.php, src/Router.php</files>
  <action>
**Create ListingController:**
Implement class with dependency injection of ListingService. Methods correspond to REST endpoints:

- create($get, $post, $files): POST /api/listings
  - Extracts POST fields (title, description, category_id, price, condition, address object)
  - Calls $listingService->createListing($fields, $userId)
  - Returns Response::success(['listing_id' => $id], 201) or Response::validationErrors()/error() on failure
  
- read($get, $post, $files, $id): GET /api/listings/:id (PUBLIC)
  - Calls $listingService->getListingById($id)
  - Returns full listing with photos, seller info, condition
  - Returns Response::error('Listing not found', 404) if not found or soft-deleted
  
- list($get, $post, $files): GET /api/listings (PUBLIC)
  - Extracts query params: category_id, status, price_min, price_max, limit (default 20), offset (default 0)
  - Calls $listingService->listAllListings($filters, $limit, $offset)
  - Returns paginated results with metadata: {listings: [...], total: N, limit, offset}
  
- update($get, $post, $files, $id): PATCH /api/listings/:id (PROTECTED)
  - Extracts PATCH fields (same as create, all optional)
  - Calls $listingService->updateListing($id, $fields, $userId)
  - Returns updated listing or 403/404/422 on error
  
- delete($get, $post, $files, $id): DELETE /api/listings/:id (PROTECTED)
  - Calls $listingService->deleteListing($id, $userId)
  - Returns {status: 'success'} or 403/404 on error
