   - Returns conversations updated since timestamp (for polling)

   - Sets updated_at automatically if not in $data
   - Or call parent if inherited

- Every query uses PDO prepared statements (? placeholders)

<task type="auto">
  <name>Task 2: Create MessageRepository with pagination and read tracking</name>
  <files>src/Repositories/MessageRepository.php</files>
  <action>
Create MessageRepository extending BaseRepository following ListingPhotoRepository pagination pattern. Implement:

1. **findByConversationId(int $conversationId, int $limit = 20, int $offset = 0): array**
   - Returns messages ordered by created_at DESC (newest first per locked decision)
   - JOINs to users table to include sender name and avatar
   - Includes sender_id, content, is_read, created_at, read_at
   - Per research: Avoid fetching messages then separately fetching sender — use JOIN
   - MUST include $this->applyDeleteFilter()

2. **findNewSince(int $conversationId, string $sinceTimestamp): array**
   - Delta query for polling: returns messages WHERE created_at > $sinceTimestamp
   - Ordered by created_at DESC (newest first)
   - JOINs to users for sender info
   - Used by ChatService.getNewMessages() for polling endpoint
   - MUST include $this->applyDeleteFilter()

3. **countUnread(int $conversationId, int $userId): int**
   - Returns COUNT of messages where is_read = FALSE AND sender_id != $userId
   - Don't count user's own messages as unread
   - Efficient COUNT(*) query with index on (conversation_id, is_read, sender_id)
   - MUST include $this->applyDeleteFilter()

4. **markConversationRead(int $conversationId, int $userId): bool**
   - Bulk operation: UPDATE all messages in conversation
   - SET is_read = TRUE, read_at = NOW() WHERE sender_id != $userId AND is_read = FALSE
   - Don't mark user's own messages as read (sender_id != $userId condition)
   - Called by ChatService.getConversationHistory() for auto-mark-on-fetch

5. **markMessageRead(int $messageId): bool**
   - Granular operation: mark single message as read
   - UPDATE messages SET is_read = TRUE, read_at = NOW() WHERE id = ?
   - Called by PATCH /api/messages/{id}/mark-read endpoint

6. **getLatestTimestamp(int $conversationId): ?string**
   - Returns MAX(created_at) for conversation
   - Used for pagination sync and polling next-timestamp response
   - Returns format: 'Y-m-d H:i:s' or null if no messages

7. **create(array $data): int**
   - Standard CRUD create
   - Expects: conversation_id, sender_id, content, is_read (default false), created_at

Notes:
- Soft-delete filtering: applyDeleteFilter() required in findByConversationId, findNewSince, countUnread, getLatestTimestamp (not in UPDATE/INSERT)
- Prepared statements in all queries
- Per research pitfall: Missing applyDeleteFilter causes soft-deleted users' messages to appear — verify all reads have it
- Per research: Index on (conversation_id, is_read, sender_id) needed for countUnread() efficiency — note in comments but don't create (DBA task)
