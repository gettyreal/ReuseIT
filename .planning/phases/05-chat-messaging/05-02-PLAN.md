Create ChatService following AuthService and ListingService patterns. Implement via constructor injection and private methods:

**Constructor:**
```php
public function __construct(
    ConversationRepository $conversationRepo,
    MessageRepository $messageRepo,
    UserRepository $userRepo
)
```

**Public Methods:**

1. **sendMessage(int $conversationId, int $senderId, string $content): array**
   - Validates message content via validateMessageContent($content)
   - Fetches conversation to verify existence and user is participant (buyer_id or seller_id)
   - Throws InvalidArgumentException if conversation not found or user not participant
   - Persists message: $this->messageRepo->create([...])
   - Updates conversation: last_message_at, unread flag for OTHER user, updated_at
   - Determine recipient: opposite of senderId (buyer vs seller)
   - Set unread flag: if sender is buyer, set unread_by_seller=TRUE; if seller, set unread_by_buyer=TRUE
   - Returns: ['id' => $messageId, 'created_at' => date('Y-m-d H:i:s')]
   - Per locked decision: Plain text only, validates before save

2. **getConversationHistory(int $conversationId, int $userId, int $limit = 20, int $offset = 0): array**
   - Verifies user is participant (buyer_id or seller_id matches $userId)
   - Throws InvalidArgumentException if unauthorized
   - Fetches messages via $this->messageRepo->findByConversationId()
   - AUTO-MARKS all returned messages as read: $this->messageRepo->markConversationRead($conversationId, $userId)
   - Updates conversation unread flag via $this->conversationRepo->markConversationRead()
   - Gets updated unread count: $this->messageRepo->countUnread()
   - Gets latest timestamp for polling: $this->messageRepo->getLatestTimestamp()
   - Returns: ['messages' => [...], 'unread_count' => N, 'conversation_updated_at' => timestamp]
   - Per locked decision: Auto-mark-on-fetch is CRITICAL

3. **getUserConversations(int $userId, int $limit = 20, int $offset = 0): array**
   - Simple wrapper: returns $this->conversationRepo->findByUserId($userId, $limit, $offset)
   - No business logic, just delegation

4. **getNewMessages(int $conversationId, int $userId, string $sinceTimestamp): array**
   - Verifies user is participant
   - Throws InvalidArgumentException if unauthorized
   - Fetches new messages via $this->messageRepo->findNewSince($conversationId, $sinceTimestamp)
   - AUTO-MARKS returned messages as read (same as getConversationHistory)
   - Gets updated unread count
   - Gets latest timestamp for next poll
   - Returns: ['messages' => [...], 'unread_count' => N, 'conversation_updated_at' => timestamp]
   - Per locked decision: Delta polling endpoint support

5. **markMessageRead(int $messageId, int $userId): bool**
   - Fetches message with its conversation: SELECT m.*, c.buyer_id, c.seller_id FROM messages m JOIN conversations c WHERE m.id = ?
   - Verifies user is participant in the conversation (buyer_id or seller_id matches $userId)
   - Throws InvalidArgumentException if not participant (403 Forbidden in controller)
   - Calls $this->messageRepo->markMessageRead($messageId)
   - Returns bool success
   - Used by PATCH /api/messages/{id}/mark-read endpoint

**Private Method:**

**validateMessageContent(string $content): void**
Per locked decisions: Plain text only, 1-1000 chars, no empty/whitespace-only
   - $trimmed = trim($content)
   - if (empty($trimmed)) throw InvalidArgumentException('Message cannot be empty')
   - if (strlen($content) > 1000) throw InvalidArgumentException('Message exceeds 1000 character limit')
   - if ($content !== strip_tags($content)) throw InvalidArgumentException('HTML not supported')
   - Per research: 5 simple checks, inline is appropriate (don't over-engineer validation class)

**Error Handling:**
- Throw InvalidArgumentException for validation failures (controller maps to 400)
- Throw InvalidArgumentException for authorization failures (controller maps to 403)
- Throw Exception for system errors (controller maps to 500)

**Notes:**
- All timestamps in 'Y-m-d H:i:s' format (use date() function)
- No external dependencies (no validation libraries)
- Follow ListingService pattern: validate → fetch context → persist → update state → return
- Per research: Auto-mark-on-fetch is critical for unread tracking to work correctly
- Per research pitfall: Verify user is participant BEFORE marking as read to prevent cross-conversation marking
