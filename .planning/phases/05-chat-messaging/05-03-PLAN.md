  <action>
Create ChatController following UserController and ListingController patterns. Implement:

**Constructor:**
```php
public function __construct(ChatService $chatService)
```
Inject ChatService via Router dependency injection (same pattern as UserController, ListingController).

**Public Methods (HTTP Endpoints):**

1. **getConversations(array $get, array $post, array $files, array $params): string**
   HTTP: GET /api/conversations
   Query params: limit (1-50, default 20), offset (0-N, default 0)
   
   - Validate: limit in range [1, 50]
   - Get userId from $_SESSION['user_id'], return 401 if missing
   - Call $this->chatService->getUserConversations($userId, $limit, $offset)
   - Return Response::success($conversations, 200)
   - On exception: catch, return Response::error('Server error', 500)
   
   Response format (per OpenCode discretion):
   ```json
   {
     "status": "success",
     "data": [
       {
         "id": 1,
         "listing_title": "iPhone 15",
         "other_user_name": "Alice",
         "other_user_avatar": "...",
         "unread_count": 3,
         "last_message_at": "2026-03-30 15:30:00",
         "updated_at": "2026-03-30 15:30:00"
       }
     ],
     "message": "Success"
   }
   ```

2. **getConversationMessages(array $get, array $post, array $files, array $params): string**
   HTTP: GET /api/conversations/{id}/messages
   URL param: id (conversation_id)
   Query params: limit (1-50, default 20), offset (0-N, default 0)
   
   - Validate: conversationId is int, limit in [1, 50]
   - Get userId from $_SESSION, return 401 if missing
   - Call $this->chatService->getConversationHistory($conversationId, $userId, $limit, $offset)
   - Include X-Poll-Interval header in response (suggest 3000 milliseconds per research)
   - Return Response::success(['messages' => [...], 'unread_count' => N, 'conversation_updated_at' => timestamp], 200)
   - On InvalidArgumentException: return Response::error('Unauthorized', 403)
   - On other exception: return Response::error('Server error', 500)

3. **getNewMessages(array $get, array $post, array $files, array $params): string**
   HTTP: GET /api/conversations/{id}/messages/new
   URL param: id (conversation_id)
   Query params: since (ISO 8601 or 'Y-m-d H:i:s' timestamp, REQUIRED)
   
   - Validate: conversationId is int, since is present and valid timestamp format
   - Get userId from $_SESSION, return 401 if missing
   - Call $this->chatService->getNewMessages($conversationId, $userId, $since)
   - Include X-Poll-Interval header
   - Return Response::success(['messages' => [...], 'unread_count' => N, 'conversation_updated_at' => timestamp], 200)
   - On InvalidArgumentException: return Response::error('Unauthorized', 403)
   - On other exception: return Response::error('Server error', 500)

4. **sendMessage(array $get, array $post, array $files, array $params): string**
   HTTP: POST /api/conversations/{id}/messages
   URL param: id (conversation_id)
   Body (JSON): { "content": "message text" }
   
   - Validate: conversationId is int, content is string and present
   - Get userId from $_SESSION, return 401 if missing
   - Call $this->chatService->sendMessage($conversationId, $userId, $content)
   - Return Response::success(['id' => $messageId, 'created_at' => timestamp], 201)
   - On InvalidArgumentException (validation or auth): return Response::validationErrors([['field' => 'content', 'message' => 'Message invalid']], 400) for validation, Response::error('Unauthorized', 403) for auth
   - On exception: return Response::error('Server error', 500)

5. **markConversationRead(array $get, array $post, array $files, array $params): string**
   HTTP: PATCH /api/conversations/{id}/mark-read
   URL param: id (conversation_id)
   Body: empty or {}
   
   - Validate: conversationId is int
   - Get userId from $_SESSION, return 401 if missing
   - Call $this->chatService->getConversationHistory($conversationId, $userId, limit=1) to trigger auto-mark
   - Or call repository directly: $this->conversationRepo->markConversationRead($conversationId, $userId)
   - Return Response::success(['status' => 'marked_read'], 200)
   - On InvalidArgumentException: return Response::error('Unauthorized', 403)
   - On exception: return Response::error('Server error', 500)

6. **markMessageRead(array $get, array $post, array $files, array $params): string**
   HTTP: PATCH /api/messages/{id}/mark-read
   URL param: id (message_id)
   Body: empty or {}
   
   - Validate: messageId is int
   - Get userId from $_SESSION, return 401 if missing
   - Call $this->chatService->markMessageRead($messageId, $userId)
   - Return Response::success(['status' => 'marked_read'], 200)
   - On InvalidArgumentException: return Response::error('Unauthorized', 403)
   - On exception: return Response::error('Server error', 500)

**Error Handling Pattern (follow UserController):**
- 400: Validation errors (missing required fields, invalid types, out-of-range)
- 401: Unauthenticated (no session user_id)
- 403: Unauthorized (not conversation participant, not message sender/recipient)
- 500: Server errors (database, unexpected exceptions)

**Response Envelope (all responses use Response class):**
- Success: Response::success($data, $httpCode)
- Validation error: Response::validationErrors([$field, $message], $httpCode)
- Error: Response::error($message, $httpCode)

**Notes:**
- Per research: Polling interval default 3000ms (3 seconds) in X-Poll-Interval header
- Per locked decision: Auto-mark happens in ChatService, not controller
- Per locked decision: Plain text only (validation in service)
- Per research: Avoid fetching full history when only marking read — optimizable with direct repository call

   - Endpoint: ChatController::markConversationRead
   - Middleware: AuthMiddleware
   - Method: PATCH
