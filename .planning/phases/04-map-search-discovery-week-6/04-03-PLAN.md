# Phase 04-03: Search Endpoint Integration & Response Formatting

## Objective

Integrate distance-based search service into GET /api/listings/search endpoint, add parameter validation, and format response with distance information. This plan completes the Phase 4 backend by connecting all layers.

## Context

- **Phase 3 deliverables:** GET /api/listings/search skeleton endpoint (exists but without distance)
- **Phase 4-01 deliverables:** GeometryService.haversineDistance(), ListingRepository.searchCandidatesByFilters()
- **Phase 4-02 deliverables:** ListingService.searchWithDistance(), location fallback
- **Phase 4 decisions locked:** Optional lat/lng with user profile fallback, 50km cap, keyword search on title+description
- **Out of scope:** Frontend map UI, WebSocket real-time updates

## Tasks

### Task 1: Enhance ListingController.search() with Distance Parameters

**Name:** Update GET /api/listings/search endpoint to handle spatial parameters

**Files:** `src/Controller/ListingController.php` (MODIFY)

**Action:**

Enhance the existing `search()` method to support distance-based discovery:

1. **Method location:** Modify existing `public function search(array $get, array $post, array $files, array $params): string`

2. **Extract and validate query parameters:**
   ```php
   // Extract parameters
   $lat = isset($get['lat']) ? (float)$get['lat'] : null;
   $lng = isset($get['lng']) ? (float)$get['lng'] : null;
   $radius = isset($get['radius']) ? (int)$get['radius'] : 10000; // Default 10km in meters
   $keyword = $get['keyword'] ?? null;
   $category_id = isset($get['category_id']) ? (int)$get['category_id'] : null;
   $condition = $get['condition'] ?? null;
   $price_min = isset($get['price_min']) ? (float)$get['price_min'] : null;
   $price_max = isset($get['price_max']) ? (float)$get['price_max'] : null;
   $limit = isset($get['limit']) ? min((int)$get['limit'], 100) : 20; // Max 100 per page
   $offset = isset($get['offset']) ? (int)$get['offset'] : 0;
   ```

3. **Validate user authentication:**
   - Check `$_SESSION['user_id']` is set
   - If not authenticated and lat/lng not provided, return 401 error: "Authentication required when using default location"
   - If authenticated, location fallback available (service will handle)

4. **Validate input parameters:**
   - `$radius`: must be integer 0-50000
     - If invalid: return 400 error "Radius must be integer between 0 and 50000 meters"
   - `$lat`: if provided, must be valid float -90 to 90
     - If invalid: return 400 error "Latitude must be between -90 and 90"
   - `$lng`: if provided, must be valid float -180 to 180
     - If invalid: return 400 error "Longitude must be between -180 and 180"
   - Keyword length: if provided, must be 1-100 characters
     - If too long: truncate to 100 characters or return 400 (choose one)

5. **Build filters array:**
   ```php
   $filters = [];
   if ($keyword !== null) $filters['keyword'] = $keyword;
   if ($category_id !== null) $filters['category_id'] = $category_id;
   if ($condition !== null) $filters['condition'] = $condition;
   if ($price_min !== null) $filters['price_min'] = $price_min;
   if ($price_max !== null) $filters['price_max'] = $price_max;
   ```

6. **Call service method with error handling:**
   ```php
   try {
       $result = $this->listingService->searchWithDistance(
           $lat,
           $lng,
           $radius,
           $filters,
           $limit,
           $offset
       );
   } catch (InvalidArgumentException $e) {
       return Response::error($e->getMessage(), 400);
   } catch (Exception $e) {
       return Response::error('Search failed: ' . $e->getMessage(), 500);
   }
   ```

7. **Format response (detailed below in Task 2):**
   - Ensure distance_meters is included for each listing
   - Include search metadata

**Verify:**

- [ ] Method extracts lat, lng, radius from query parameters
- [ ] Default values: lat/lng = null (fallback), radius = 10000m
- [ ] Radius validation rejects < 0 and > 50000
- [ ] Lat/lng validation enforces -90/90 and -180/180 bounds
- [ ] Keyword validation enforces length limit
- [ ] Filters array built correctly
- [ ] Service method called with correct parameters
- [ ] Exceptions caught and return appropriate HTTP status codes (400 for validation, 401 for auth, 500 for server errors)

**Done:**

- Call `/api/listings/search?lat=43.65&lng=-79.38&radius=5000`: searches 5km radius
- Call `/api/listings/search?keyword=iPhone`: uses user's stored location (fallback)
- Call `/api/listings/search?radius=60000`: returns 400 "Radius must be... 50000"
- Call `/api/listings/search` unauthenticated without lat/lng: returns 401
- Call with category_id, price filters: all filters combined (AND logic)

**Files:** `src/Controller/ListingController.php` (MODIFY)

**Action:**

Format the service response into JSON envelope with distance information:

1. **Transform listing data:**
   - For each listing returned from service, ensure it includes:
     - Standard fields: `id`, `title`, `description`, `price`, `category_id`, `status`, `created_at`
     - Location fields: `latitude`, `longitude`
     - Distance field: `distance_meters` (from service)

## Verification

### API Contract
- **Endpoint:** `GET /api/listings/search`
- **Query parameters:**
  - `lat` (optional float, -90 to 90)
  - `lng` (optional float, -180 to 180)
  - `radius` (optional int meters, 0-50000, default 10000)
  - `keyword` (optional string, 1-100 chars)
  - `category_id` (optional int)
  - `condition` (optional string enum)
  - `price_min`, `price_max` (optional floats)
  - `limit` (optional int, 1-100, default 20)
  - `offset` (optional int, default 0)
- **Success response:** HTTP 200, `{"status": "success", "data": {"listings": [...], "search": {...}}}`
- **Error responses:**
  - 400: validation error (radius > 50km, invalid lat/lng)
  - 401: authentication required (no lat/lng provided, user not logged in)
  - 500: server error

### Integration Points
- ListingController.search() calls ListingService.searchWithDistance()
- ListingService calls ListingRepository.searchCandidatesByFilters()
- ListingService uses GeometryService.haversineDistance()
- ListingService accesses UserRepository for location fallback
- Response envelope follows Phase 1 standard format

### Backward Compatibility
- Existing list endpoints (GET /api/listings) unchanged
- Existing create/update/delete endpoints unchanged
- Only GET /api/listings/search enhanced (new parameters, new fields)
