## Tasks

### Task 1: Create GeometryService with Haversine Implementation

**Name:** Implement GeometryService with static Haversine method

**Files:** `src/Utils/GeometryService.php` (CREATE)

**Action:**

Create a new utility class `GeometryService` with static method `haversineDistance()`:

1. Define constant `EARTH_RADIUS_KM = 6371`
2. Implement Haversine formula:
   - Accept parameters: `$lat1, $lng1, $lat2, $lng2` (floats, in decimal degrees)
   - Convert degrees to radians using `deg2rad()`
   - Calculate differences: `$dLat = deg2rad($lat2 - $lat1)`, `$dLng = deg2rad($lng2 - $lng1)`
   - Apply formula: `$a = sin($dLat/2)² + cos($lat1 rad) × cos($lat2 rad) × sin($dLng/2)²`
   - Apply formula: `$c = 2 × atan2(√a, √(1-a))`
   - Return: `EARTH_RADIUS_KM × $c` (distance in kilometers)
3. Add public PHPDoc with parameter types, return type (float), units documented

**Note:** Class uses only built-in PHP math functions (`sin`, `cos`, `deg2rad`, `atan2`, `sqrt`). No external dependencies.

**Verify:**

- [ ] File exists at `src/Utils/GeometryService.php`
- [ ] Method signature: `public static function haversineDistance(float $lat1, float $lng1, float $lat2, float $lng2): float`
- [ ] Return value is in kilometers (positive float)
- [ ] No syntax errors (PHP can parse file)

**Done:**

- Toronto (43.6532, -79.3832) to Montreal (45.5017, -73.5673) returns ~330 km
- Same point returns ~0 km (not exactly 0 due to float precision, but < 1 meter)
- Uses standard EARTH_RADIUS_KM = 6371 (widely-used constant)

### Task 2: Add searchCandidatesByFilters() to ListingRepository

**Name:** Enhance ListingRepository with distance-ready candidate query method

**Files:** `src/Repository/ListingRepository.php` (MODIFY)

**Action:**

Add new method `searchCandidatesByFilters()` to ListingRepository:

1. **Method signature:**
   ```php
   public function searchCandidatesByFilters(
       array $filters = [],
       int $limit = 1000
   ): array
   ```

2. **Parameters:**
   - `$filters`: associative array with optional keys:
     - `keyword`: string (partial match in title OR description)
     - `category_id`: int or null
     - `condition`: string or null (enum: 'new', 'like_new', 'good', 'fair')
     - `price_min`: float or null
     - `price_max`: float or null
   - `$limit`: max results to return (default 1000 for distance calc)

3. **Query logic:**
   - SELECT: listings with seller user info + photo count
   - JOIN: users (to get seller details)
   - LEFT JOIN: photos (to count)
   - WHERE: 
     - `listings.deleted_at IS NULL` (soft-delete filter)
     - `users.deleted_at IS NULL` (seller must not be deleted)
     - `listings.status = 'active'` (only active listings)
     - `listing_photos.deleted_at IS NULL` (soft-delete on photos)
   - Optional WHERE conditions:
     - If `keyword` provided: `(listings.title LIKE ? OR listings.description LIKE ?)`
     - If `category_id` provided: `listings.category_id = ?`
     - If `condition` provided: `listings.condition = ?`
     - If `price_min` provided: `listings.price >= ?`
     - If `price_max` provided: `listings.price <= ?`
   - GROUP BY: `listings.id` (to handle photo count aggregation)
   - ORDER BY: `listings.created_at DESC` (consistent with existing behavior)
   - LIMIT: `$limit` (for distance calc, don't paginate here)

4. **Result format:**
   - Each row must include: `id, latitude, longitude, title, price, category_id, seller_id, status, created_at, photo_count`
   - Return array of associative arrays (one per row)

5. **Use prepared statements:**
   - All `$filter` values must be bound as placeholders (`?`)
   - Never interpolate `$filter` values directly into SQL

**Implementation notes:**
   - Reuse existing WHERE condition patterns from Phase 3's `filterCombined()` method
   - Do NOT paginate results (service layer handles pagination after distance sort)
   - Do NOT filter by distance in SQL (that's done in PHP)
   - Ensure soft-delete filtering matches pattern used in other repository methods

**Verify:**

- [ ] Method exists and is public
- [ ] All filter values use prepared statement placeholders
- [ ] Query returns results with `latitude` and `longitude` fields
- [ ] Result set includes photo count (for frontend rendering)
- [ ] Soft-delete filters applied to listings, users, photos
- [ ] No results returned if seller is soft-deleted
- [ ] No results returned if listing is soft-deleted

**Done:**

- Call with empty filters: returns all active listings
- Call with `keyword='iPhone'`: returns listings where title OR description contains 'iPhone'
- Call with `category_id=2, price_min=100, price_max=500`: returns electronics 100-500
- Combined filters work: keyword + category + price all applied (AND logic)
- Result order consistent (newest first)
