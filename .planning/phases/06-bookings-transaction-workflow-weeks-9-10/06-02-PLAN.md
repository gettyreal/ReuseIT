- `createBooking(int $listingId, int $buyerId): array`
  - Begin transaction.
  - Lock listing and active booking scope (`FOR UPDATE`).
  - Enforce: listing active, buyer != seller, no active pending/confirmed booking.
  - Insert booking with status `pending`, `expires_at = NOW()+12h` (locked decision).

  - Lock booking row and require status `pending`, actor is seller, not expired.
  - Transition to `confirmed`, create `confirmed` event.

  - Only participants, only from `confirmed`.
  - Transition to `completed`, set `completed_at`, create `completed` event.
- `cancelBooking(int $bookingId, int $actorUserId, string $reasonCode, ?string $note): array`

  - Transition to `cancelled`, set cancellation metadata, create `cancelled` event with reason/timestamp.

  - Find overdue pending bookings (`expires_at <= NOW()`), transition to cancelled reason `expired`, append `expired` event, call notification service for both sides.
