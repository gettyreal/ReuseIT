3. Cache strategy:
   - Input: address array with 'street', 'city', 'province', 'postal_code', 'country' keys
   - Step 1: Normalize address string: "{street}, {city}, {province}, {postal_code}, {country}"
   - Step 2: Generate cache key: md5($normalizedAddress)

   - Step 7: Return coordinates or null if API fails/no results

API URL pattern: https://maps.googleapis.com/maps/api/geocode/json?address={encoded_address}&key={GOOGLE_MAPS_API_KEY}

Error handling: Wrap API call in try-catch; return null on any failure (API down, rate limit, invalid address); log failures (optional per OpenCode discretion)

Permission: Use $_ENV['GOOGLE_MAPS_API_KEY'] or getenv('GOOGLE_MAPS_API_KEY'); fail gracefully if not set

<task type="auto">
  <name>task 3: Create AuthService with registration, login, logout logic using password hashing and session management</name>
  <files>src/Services/AuthService.php</files>
  <action>
Create src/Services/AuthService.php implementing core authentication business logic.

Constructor: accepts:
- private UserRepository $userRepo
- private GeolocationService $geoService
- private SessionHandler $session (from Phase 1)

Public methods:

1. register(string $email, string $password, string $firstName, string $lastName, array $address, ?array $coordinates = null): array
   - Validate email uniqueness: if (findByEmail($email)) throw Exception('Email already registered')
   - Hash password: password_hash($password, PASSWORD_BCRYPT, ['cost' => 12])
   - Geocode address if $coordinates not provided: $coordinates = $geoService->geocodeAddress($address); if (!$coordinates) throw Exception('Unable to geocode address...')
   - Create user: $userId = $userRepo->create([...all fields...])
   - Create session: $this->session->login($userId)
   - Return ['user_id' => $userId] (session cookie set automatically by SessionHandler)

2. login(string $email, string $password): array
   - Rate limiting check: (will be in Plan 02, for now assume it passes)
   - Find user: $user = $userRepo->findByEmail($email)
   - If not found: throw Exception('Invalid credentials') [prevents email enumeration]
   - Verify password: if (!password_verify($password, $user['password_hash'])) throw Exception('Invalid credentials')
   - Create session: $this->session->login($user['id']) [regenerates session ID for security]
   - Return ['user_id' => $user['id']]

3. logout(): void
   - Get session ID from cookie: $sessionId = $_COOKIE['PHPSESSID'] ?? null
   - If exists: $this->session->logout($sessionId) [clears session from database and cookie]

4. getCurrentUser(): ?array
   - Check $_SESSION['user_id']; if not set return null
   - Return $userRepo->find($_SESSION['user_id'])

All password operations use password_hash() and password_verify() (no custom hashing).
All database operations via $userRepo (no direct PDO calls).
All errors thrown as Exception with user-friendly messages (no stack traces).

Test: Register → login with correct password → verify session created. Login with wrong password → verify Exception thrown.

Methods:

1. register(array $get, array $post, array $files, array $params): string
   - Endpoint: POST /api/auth/register
   - Parse JSON request body: json_decode(file_get_contents('php://input'), true)
   - Required fields: email, password, first_name, last_name, address (array with 5 components), coordinates (optional)
   - Validate:
     - email: non-empty, valid format (use Respect\Validation from Phase 1)
     - password: non-empty, min 8 chars
     - first_name, last_name: non-empty, <100 chars
     - address: all 5 components non-empty (street, city, province, postal_code, country)
   - Call AuthService.register(...)
   - On success: return Response::success(['user_id' => ...], 'Registration successful') with HTTP 201
   - On Exception: return Response::error(exception message, 400) [e.g., 'Email already registered']
   - Catch all other errors: return Response::error('Server error', 500)

2. login(array $get, array $post, array $files, array $params): string
   - Endpoint: POST /api/auth/login
   - Parse JSON request body
   - Required fields: email, password
   - Validate: email non-empty, password non-empty
   - Call AuthService.login(...)
   - On success: return Response::success(['user_id' => ..., 'session_id' => session_id()], 'Login successful') with HTTP 200
   - On Exception: return Response::error('Invalid credentials', 401)

3. logout(array $get, array $post, array $files, array $params): string
   - Endpoint: POST /api/auth/logout
   - Call AuthService.logout()
   - Return Response::success([], 'Logged out successfully') with HTTP 200
   - No authentication required (user might be already logged out)

4. me(array $get, array $post, array $files, array $params): string
   - Endpoint: GET /api/auth/me
   - Requires authentication: if (empty($_SESSION['user_id'])) return Response::error('Unauthorized', 401)
   - Get current user: $user = AuthService.getCurrentUser()
   - Return Response::success($user, 'User retrieved') with HTTP 200

- GET /api/auth/me → AuthController.me
