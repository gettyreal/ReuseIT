  <action>
Create src/Middleware/AuthMiddleware.php implementing authentication enforcement.

Class structure:

```php
class AuthMiddleware {
    /**
     * Require authentication.
     * If user not logged in (no $_SESSION['user_id']), throw exception.
     * Called by Router before executing protected endpoint actions.
     * 
     * @throws Exception if not authenticated
     */
    public function requireAuth(): void {
        if (empty($_SESSION['user_id'])) {
            throw new Exception('Unauthorized');
        }
    }
    
    /**
     * Check if user is authenticated.
     * Returns true if $_SESSION['user_id'] exists, false otherwise.
     */
    public function isAuthenticated(): bool {
        return !empty($_SESSION['user_id']);
    }
    
    /**
     * Get current user ID.
     * Must call requireAuth() first to ensure user is logged in.
     */
    public function getCurrentUserId(): int {
        return (int)($_SESSION['user_id'] ?? 0);
    }
}
```

Integration with Router.php:

In Router.dispatch() method, before calling controller action:
1. Check if endpoint is protected (identify by controller + action name)
2. If protected: create AuthMiddleware instance and call requireAuth()
3. If requireAuth() throws: catch and return Response::error('Unauthorized', 401)
4. If passes: continue with controller action

Protected endpoints list (grows over time):
- UserController: show() = public, update() = protected
- AuthController: register() = public, login() = public, logout() = public, me() = protected

Example dispatch logic:
```php
$protected = [
    'UserController:update',
    'AuthController:me',
    // Phase 3+: ListingController:create, ListingController:update, etc.
];

if (in_array("{$controller}:{$action}", $protected)) {
    $middleware = new AuthMiddleware();
    try {
        $middleware->requireAuth();
    } catch (Exception $e) {
        return Response::error('Unauthorized', 401);
    }
}
```

Error responses for authentication:
- If authentication required but missing: HTTP 401 Unauthorized with message "Unauthorized"
- If authorization failed (e.g., user edits someone else's profile): HTTP 403 Forbidden with message "Forbidden"

All protected endpoints must check authentication BEFORE processing request.
All protected endpoints that modify resources must check authorization (resource ownership).

Test: PATCH /api/users/1/profile without session → 401. PATCH with session → 200.
