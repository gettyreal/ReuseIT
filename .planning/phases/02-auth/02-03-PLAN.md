<objective>
Implement rate limiting to prevent brute-force login attacks and secure geolocation caching. Track failed login attempts per email + IP; lock account for 15 minutes after 5 failures. This hardens authentication against common attack vectors.

**Purpose:**
- Prevent brute-force attacks on user accounts (dictionary/password list attacks)
- Discourage enumeration attacks (no way to determine if email exists via response time)
- Cache geolocation results to avoid wasting Google Maps API quota
- Locked decisions from CONTEXT.md: "5 failed attempts → 15-minute lock" and "Email notification when locked" (deferred to implementation)

**Output:**
- RateLimitService with isLocked(), recordFailedAttempt(), clearAttempts() methods
- LoginAttemptRepository for database access
- login_attempts table with email, ip_address, attempt_count, locked_until columns
- Integration in AuthService.login() to check rate limit before validation
- Integration to clear counter on successful login
</objective>

<task type="auto">
  <name>task 2: Create RateLimitService with failed attempt tracking and lockout logic</name>
  <files>src/Services/RateLimitService.php</files>
  <action>
Create src/Services/RateLimitService.php implementing rate limiting.

Class constants:
```php
private const MAX_ATTEMPTS = 5;        // Lock after 5 failed attempts
private const LOCKOUT_MINUTES = 15;    // Lock duration in minutes
```

Constructor: accepts:
- private LoginAttemptRepository $attemptRepo

Public methods:

1. isLocked(string $email, string $ipAddress): bool
   - Check if (email, ip_address) pair is currently locked
   - Query: $row = attemptRepo->find($email, $ipAddress)
   - If row exists and locked_until > NOW(): return true
   - Else: return false
   - This prevents login attempts while locked

2. recordFailedAttempt(string $email, string $ipAddress): void
   - Called on failed login (wrong email or wrong password)
   - Increment attempt_count for this email+ip pair
   - If attempt_count >= MAX_ATTEMPTS: set locked_until = NOW() + 15 minutes
   - Steps:
     a) Query existing row: $row = attemptRepo->find($email, $ipAddress)
     b) If row exists:
        - increment attempt_count
        - if new count >= MAX_ATTEMPTS: set locked_until = DATE_ADD(NOW(), INTERVAL ? MINUTE) [15]
        - call attemptRepo->update($email, $ipAddress, ['attempt_count' => new_count, 'locked_until' => locked_until])
     c) If row doesn't exist: attemptRepo->create(['email' => $email, 'ip_address' => $ipAddress, 'attempt_count' => 1])

3. clearAttempts(string $email, string $ipAddress): void
   - Called on successful login
   - Delete the row: attemptRepo->delete($email, $ipAddress)
   - This resets the counter for next attempt cycle

4. cleanup(): void (optional, can be deferred)
   - Call attemptRepo->clearOlderThan(LOCKOUT_MINUTES) to remove old lockout records
   - Can be called periodically (e.g., from a cron job)

Error message for locked account:
- Return message: "Too many login attempts. Try again in 15 minutes."
- Do not reveal specific counts or exact unlock time (prevent user enumeration)

All operations use LoginAttemptRepository (no direct PDO).

Test: Record 5 failed attempts → verify locked_until set. recordFailedAttempt 6th time → verify still locked (not incrementing). clearAttempts() → verify row deleted.

Modify the exception handling in login():

```php
public function login(array $get, array $post, array $files, array $params): string {
    try {
        // ... existing validation code ...
        
        // Call AuthService.login()
        $result = $this->authService->login($email, $password);
        
        return $this->response->success(
            ['user_id' => $result['user_id'], 'session_id' => session_id()],
            'Login successful'
        );
    } catch (Exception $e) {
        // Both "Too many login attempts" and "Invalid credentials" return same HTTP 401
        // This prevents attackers from distinguishing between lockout and bad credentials
        
        $message = $e->getMessage();
        
        if (strpos($message, 'Too many login attempts') !== false) {
            return $this->response->error($message, 429); // 429 Too Many Requests (optional; 401 is also acceptable)
        } else if (strpos($message, 'Invalid credentials') !== false) {
            return $this->response->error($message, 401);
        } else {
            return $this->response->error('Server error', 500);
        }
    }
}
```

Alternatively, return 401 for both lockout and invalid credentials (simpler, same security):
```php
return $this->response->error($e->getMessage(), 401);
```

Keep error messages generic ('Invalid credentials', 'Too many login attempts') — no details about which field failed.
